# Ecommerce-Order-Management
This repository contains an end-to-end Business Analysis project for a web-based E-commerce Order Management and Checkout System. The project demonstrates the complete Business Analyst lifecycle, from understanding business problems to defining system requirements and supporting Agile delivery. 
The objective of this project is to improve the customer shopping experience by optimizing checkout flow, order tracking, and return management, while providing administrators with better visibility and control over orders.

# Business Problem
The business faces challenges such as cart abandonment, inefficient order processing, and limited visibility into order and return statuses. This project addresses these issues by defining clear requirements, user journeys, and system behavior for a scalable e-commerce platform.

# Project Scope
In Scope
User registration and authentication
Product browsing and product details
Cart and checkout process
Order tracking and order history
Returns and refund management
Admin order management

Out of Scope
Mobile application
International shipping
Vendor onboarding

# Artifacts Included

Business Requirements Document (BRD) – Business objectives, scope, and high-level requirements
Software Requirements Specification (SRS) – Functional and non-functional system requirements
Functional Requirements Document (FRD) – Business rules, validations, and edge cases
User Stories & Acceptance Criteria – Agile-ready requirements
Process Flow Diagrams – Checkout, order fulfillment, and return workflows
Low-Fidelity Wireframes – Customer and admin screens
Jira Agile Simulation – Epics, user stories, sprint planning, and board tracking
